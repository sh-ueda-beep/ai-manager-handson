記録済みのハンズオンステップをすべて一覧表示する。

---

## 動作フロー

### 1. ステップファイルの取得

`handson/steps/` ディレクトリ内の `step-*.md` ファイルを検索する。

```bash
ls -1 handson/steps/step-*.md 2>/dev/null || echo "NO_STEPS"
```

### 2. 索引ファイルの読み取り

`handson/_index.yaml` を読み取り、登録済みステップの情報を取得する。

### 3. 各ステップの情報を抽出

各ステップファイルのフロントマター（YAML）から以下を取得：
- `step`: ステップ番号
- `title`: タイトル
- `status`: ステータス（draft / review / final）
- `created`: 作成日時
- `tags`: タグ

### 4. 一覧を表示

**ステップが存在する場合：**

```
## ハンズオン ステップ一覧

| # | タイトル | ステータス | 作成日 | タグ |
|---|---------|-----------|--------|------|
| 1 | OpenSpecの初期化 | draft | 2026-02-27 | setup, openspec |
| 2 | ... | ... | ... | ... |

合計: {N} ステップ（draft: {n1}, review: {n2}, final: {n3}）
```

**ステップが存在しない場合：**

```
## ハンズオン ステップ一覧

まだステップが記録されていません。

作業を行った後に `/handson:record` でステップを記録してください。
```

### 5. 整合性チェック

`_index.yaml` とファイルシステムの間に不整合がある場合（索引にあるがファイルがない、またはその逆）、警告を表示する。

```
⚠ 整合性の警告:
- 索引に存在するがファイルがないステップ: step-03-xxx
- ファイルが存在するが索引にないステップ: step-04-yyy
```

---

## ガードレール

- 読み取り専用の操作。ファイルの変更は行わない
- すべてのテキストは**日本語**で表示する
